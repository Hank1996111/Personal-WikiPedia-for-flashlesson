## 這堂課學了什麼

### Minerva 學習理論框架
- **遷移學習**：不只是記答案，而是理解原理後能應用到陌生情境（例：學統計學原理 → 能判斷政治民調可信度）
- **認知科學三件事**：注意力（濾網）、記憶（倉庫，需反覆提取）、先備知識（掛鉤，新知識需連結已知才好記）
- **HC vs FC**：HC 是「習慣性思考」（怎麼想的工具，需練習內化）；FC 是「基礎概念」（跨學科框架，可被學習與應用）
- 兩者天生同時學習：學 FC 的過程中採取 HC 的思維方式

### 學習 FC 的五個問題
1. 核心機制是什麼？
2. 邊界條件（什麼時候成立/不成立）？
3. 跨域舉三個例子
4. 連結真實生活中的問題
5. 用比喻教給別人（30 秒內）

## #assertions 實戰練習

### 核心定義
任何論述由三部分組成：**表面前提 + 結論 + 隱藏前提**

隱藏前提是「必須成立才能讓結論站得住腳，但沒有被說出來的假設」。

### 關鍵洞察
- 隱藏前提是中性工具：可用來分析（接收者），也可刻意使用（發出者，如領導者建立團隊共同前提）
- **共鳴感不是證明**，它只是讓你降低了對隱藏前提的懷疑
- 最常見的邏輯跳躍：「A 和 B 同時出現 → A 造成 B」（相關性被當成因果）→ 要問有沒有第三個變因 C 同時造成 A 和 B（混淆變項）

### 練習案例
- 「吃早餐學生成績較好 → 學校應提供免費早餐提升成績」：隱藏假設是相關性等於因果；真正的混淆變項是家庭資源
- 「AI 越來越強 → 學技術性技能沒有意義」：隱藏假設是「AI 能做的事人類就沒有價值做」→ 把價值定義得太窄了
- 課程規劃 WHY 文案：建立「市場上大多數業務都有這個痛點」的隱藏前提，讓讀者自動對號入座

## #critique 實戰練習

### 四個核心問題
1. **誰在說？（Source）**：作者身分、立場、動機——沒有人是完全中立的
2. **說給誰聽？（Audience）**：預設讀者是誰，同一篇文章可能同時對多個受眾說不同的事
3. **說了什麼沒說？（Omission）**：文本選擇強調什麼就必然淡化什麼，不在場的資訊往往比在場的更重要
4. **為什麼現在說？（Context）**：時機本身就是訊息

### Omission 的本質
Omission 讓你思考「作者沒有提到的部分」，成為後續驗證或學習的動機。隱藏前提讓你認識這份文件背後的目的。差別：隱藏前提是主動植入，Omission 是主動迴避。

### 練習案例：日本稅務士 Claude Code 文章
- Source：會計師 + X 個人品牌創作者（兩個身分的利益不完全相同）
- Audience：顧問客戶（強調資安）+ AI 玩家（強調可複製性）+ 潛在投資人/合作者
- Omission：客戶規模沒說、出錯時的處理機制沒說、系統建立的過程與代價沒說、不懂程式的他如何 debug 沒說
- Context：AI Agent 元年，這個時間點發文傳播效果最大化

### 練習案例：講師課程規劃 doc
- 作為平台評估者的面談問題清單：
  1. 有沒有非同步教學的經驗？
  2. Module 5（個人品牌）對主力受眾（資深業務）的實際 ROI 是多少？
  3. 具體數字（提案時間從 8 小時壓縮至 2 小時）的來源是什麼？

## 個人盲點紀錄

### 在 #assertions 上
- 傾向找「目的或價值觀」層面的假設，但容易錯過邏輯跳躍本身（A→B 的因果結構）

### 在 #critique 上
- 原有 Why/What/Who 框架已強，今天新增了 Omission 和隱藏前提的視角
- Omission 讓框架從「理解一件事」升級到「質疑一件事」

### 在給建議上（Howard 活動企劃案例）
- 給了原則但沒給可操作的下一步
- 在資訊不完整的情況下（「我下面的就沒看了」）就開始下判斷
- 建議混合了兩個不同層次的問題（文件結構 vs 活動創意）
- 結論清楚，但對方進入你框架的路沒有鋪好——跟你批評的問題是同一個
